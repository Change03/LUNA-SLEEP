## 파일별 기록 항목 (권장)
- 출처 이름: 운영자가 입력
- 출처 URL 또는 원본 위치: 운영자가 입력
- 파일명(원본): 운영자가 입력
- 저장 경로(프로젝트): public/images/…
- 허락 여부(allowed/denied/needs-confirmation)
- 허락 상세(텍스트 사용 가능 여부, 상세 이미지 사용 여부 등)
- 확인자(운영자 담당자 이름)
- 허락 날짜

## 허락 범위 예시
- 제품 사진: 허용
- 브랜드 소개서 이미지: 허용
- 신세계쇼핑 루나앤슬립 상품 사진: 허용
- 상세페이지 통이미지: 루나앤슬립 상품 사진만 사용, 판매처 UI/배너 제외
- 상품 상세 문구: 자사몰 톤으로 재작성

## 주의사항
- 허락 범위가 불명확한 문구, 리뷰, 고객 후기, 타사 로고, 판매처 고유 UI 디자인은 그대로 복사하지 마세요.
- 상품 설명 문구는 가능하면 루나앤슬립 브랜드 톤에 맞게 새로 작성하세요.
- 기존 쇼핑몰 상세페이지 이미지는 허락받은 경우에만 사용하세요.
- 사용한 이미지의 출처와 허락 상태를 반드시 프로젝트 문서에 기록하세요.

## 운영 전 최종 확인 체크리스트
- [ ] 출처와 허락자(담당자) 기입됨
- [ ] 허락 날짜 기입됨
- [ ] 허락 범위(이미지 종류, 텍스트 사용 등) 명확히 기입됨
- [ ] 사용 금지 항목에 해당되지 않음 확인
- [ ] 관리자 페이지에서 permission 상태가 `approved` 인 자산만 공개됨 확인
- [ ] `assets/asset-manifest.json`에서 판매처 UI/배너/리뷰 이미지가 skipped 처리되었는지 확인
- [ ] `assets/brand-deck-assets.json`에서 브랜드 소개서 추출 이미지 경로 확인
- [ ] `assets/product-content-sources.json`에서 상품 상세정보와 FAQ 재구성 출처 확인

## 권장 워크플로우
1. 운영자로부터 자산(이미지 ZIP, 브랜드 소개서, 또는 상품 URL 목록)을 받는다.
2. `assets/permissions.json`에 출처와 허락 범위를 기록한다.
3. `assets/source-urls.json`에 허락받은 상품 URL을 추가한다.
4. `npm run import:assets -- --dry-run`로 시뮬레이션을 실행해 검토한다.
5. 실제 다운로드 시 `npm run import:assets`를 실행하고 `assets/asset-manifest.json`을 검토한다.
6. 관리자에서 자산을 확인·승인 후 상품 페이지에 노출한다.
